# Week5 Agent Simulation Homework

## 프로젝트 소개

본 프로젝트는 LLM 기반 Multi-Agent Simulation 시스템을 구현한 과제입니다.

기존의 중고거래 플랫폼(당근마켓, 번개장터)은 물건을 구매하거나 판매하는 것에 초점을 맞추고 있습니다. 하지만 실제 생활에서는 짧은 기간 동안만 필요한 물건을 위해 새 제품을 구매하거나 중고거래를 하는 것이 비효율적인 경우가 많습니다.

저는 동아리에서 진행했던 '하루만(HARUMAN)' 프로젝트를 바탕으로, 대여 중심 플랫폼에서 사람들이 어떻게 상호작용하는지를 시뮬레이션하고자 하였습니다.

예를 들어 콘서트 촬영을 위해 카메라가 필요하지만 구매는 부담스러운 사용자, 사용하지 않는 물건을 대여하여 수익을 얻고 싶은 판매자, 그리고 대여 플랫폼 창업을 준비하는 사용자가 온라인 공간에서 만나 대화하도록 설계하였습니다.

이를 통해 대여 플랫폼 환경에서 사용자들의 행동과 관계 형성 과정을 관찰하고자 하였습니다.


---

## 기획 의도

당근마켓과 번개장터는 구매 중심 플랫폼입니다.

반면 HARUMAN은 다음과 같은 상황을 해결하고자 합니다.

- 콘서트 촬영을 위해 카메라가 하루만 필요한 경우
- 캠핑을 위해 텐트를 잠시 사용하고 싶은 경우
- 자주 사용하지 않는 물건을 구매하기 부담스러운 경우

이러한 상황에서 사용자들이 물건을 구매하지 않고 대여를 선택할 수 있도록 하는 플랫폼을 가정하였습니다.

따라서 본 시뮬레이션에서도 일반 거래 커뮤니티뿐 아니라 HARUMAN 웹사이트라는 가상 공간을 추가하여 대여 중심 대화가 이루어지도록 설계하였습니다.

## 구현 내용

### Agent

각 Agent는 다음 정보를 가집니다.

- 이름
- 나이
- 직업
- 성격
- Memory
- Reflection

---

### Planning

하루 시작 전 각 Agent는 LLM을 통해 시간대별 활동 계획을 생성합니다.

예시:

- 콘서트 팬 커뮤니티
- 당근 채팅방
- 번개장터 거래방
- 대여 판매자 커뮤니티
- HARUMAN 웹사이트

---

### Conversation

같은 시간에 같은 공간에 존재하는 Agent들은 자동으로 대화를 진행합니다.

공간에 따라 대화 주제가 달라집니다.

- HARUMAN 웹사이트 → 대여 거래
- 콘서트 팬 커뮤니티 → 공연 및 가수 이야기
- 당근 채팅방 → 중고거래
- 번개장터 거래방 → 판매 및 가격 협상
- 대여 판매자 커뮤니티 → 대여 사업 및 운영

---

### Memory

대화가 끝난 후 중요한 사실(Fact)을 추출하여 Agent의 Memory에 저장합니다.

---

### Reflection

대화 내용을 바탕으로 다른 Agent에 대한 관계를 요약하여 Reflection을 생성합니다.

---

### Report

시뮬레이션 종료 후 특정 Agent 관점에서

- 알고 있는 사실
- 다른 Agent에 대한 관계

를 리포트 파일로 저장합니다.
